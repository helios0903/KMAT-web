KMAT — How It Works

A visual guide to how this Korean Meeting Translator was built, explained simply.

The Big Picture

KMAT does one thing: listens to Korean speech and shows the translation in real-time. Here's the journey of your voice through the app:

🎤
Your Voice
Korean speech
→
🧠
Deepgram AI
Converts speech to text
→
✍️
Korean Text
What you said, written out
→
🤖
Claude AI
Translates to Chinese/English
→
📄
Translation
Shown on screen

Think of it like a relay race: your voice is the baton, and each station passes it to the next until it becomes a translation on your screen.

Three Layers of the App

Every web app has three layers, like a sandwich:

Frontend (What You See)
The buttons, text, colors, and layout. Built with React + Tailwind CSS. This is like the face of the app — the part users interact with.

Files: app/page.tsx app/components/* app/globals.css
Backend (The Brain)
Handles the smart work: calling AI services, processing data. The frontend sends requests here, and the backend talks to Deepgram and Claude on your behalf.

Files: app/api/translate/route.ts app/api/summary/route.ts app/api/deepgram/route.ts
Storage (The Memory)
Saves your meeting data, bookmarks, glossary, and settings so they survive page refreshes. Uses localStorage — a simple storage built into your browser.

Files: app/lib/storage.ts app/types.ts
External AI Services
The app doesn't do AI itself — it asks two cloud services:
Deepgram Nova-2 — listens to audio, returns Korean text
Claude Haiku — reads Korean text, returns translation
Both need API keys (passwords) stored in .env.local

What Happens When You Record

Step-by-step, here's what the app does when you press "Start Recording":

1
Get microphone access — Browser asks "Can this site use your microphone?" You say yes.
2
Get Deepgram key — App asks our server for the AI password. Server returns it (never exposed in the webpage code).
3
Open live connection — App opens a WebSocket (a persistent phone line) to Deepgram's servers.
4
Stream audio — Every 250ms (4 times per second), a small chunk of audio is sent to Deepgram.
5
Receive Korean text — Deepgram sends back what it heard. First as a guess (interim), then as confirmed text (final).
6
Buffer short fragments — If the text is too short (< 4 chars), wait for more. This avoids translating single syllables.
7
Translate — Send the Korean text + glossary to our server, which calls Claude AI to translate it.
8
Display — Translation appears on screen. Auto-scrolls. Saved to localStorage so it survives page refresh.

Lessons Learned & What To Do Better Next Time

Problem: Started with wrong ASR

We first used Web Speech API (free, built into Chrome) but it had poor Korean accuracy and clipped the start of sentences. We had to migrate to Deepgram mid-project.

Next time: Research ASR options first

Before writing code, test 2-3 speech recognition services with Korean audio samples. Pick the best one from the start instead of migrating later.

Problem: API key issues wasted time

We spent debugging time on invalid API keys, stale build caches, and wrong environment configs. These aren't code bugs — they're setup issues.

Next time: Verify API keys first

Before writing any feature code, create a simple test script that confirms each API key works. Add clear error messages when keys are missing.

Problem: No database

All data lives in localStorage — if you clear browser data, everything is gone. No way to share meetings between devices or with other people.

Next time: Add a real database early

Use Supabase or Firebase from the start. This enables user accounts, cross-device sync, and meeting sharing — features that are hard to bolt on later.

Problem: Large files burn AI tokens

page.tsx grew to 470+ lines. Every time the AI assistant needed to read it, it consumed a lot of usage quota. This adds up fast.

Next time: Split early, create CLAUDE.md early

Break large files into smaller modules from the start. Create a CLAUDE.md project map on day one so the AI only reads what it needs.

Summary

KMAT is a real-time Korean meeting translator built with Next.js. It captures your voice with Deepgram, translates it with Claude AI, and displays everything live on screen. It has 6 pages, 3 API routes, and stores everything in your browser's localStorage. The whole project is about 2,000 lines of code across 25 files.
